# Roadmap M0-M2

This roadmap covers the first three delivery points for NovaOS under the portable-fabric direction.

## M0: Repository refactor for portability

Deliverables:
- portable core/platform/personality repo split
- `kernel/arch/x86_64/` placeholder lane
- `drivers/bus/pci/` placeholder lane
- `services/acceld/backends/*` and `services/memd/profiles/*`
- BootInfo v2 draft
- portable-fabric architecture notes

Acceptance:
- Spark lane still builds and validates
- generic contracts no longer require Spark-only assumptions
- future workstation/server lanes have explicit scaffolding

## M1: Spark observatory freeze

Deliverables:
- `spark-observe.efi`
- structured platform dump format
- boot menu notes for Spark UEFI
- BootInfo v2 draft updated with real observed Spark facts

Acceptance:
- boots from USB on Spark
- writes a valid report
- captures firmware tables, framebuffer state, and seed-relevant facts

## M2: Spark loader v2

Deliverables:
- stage0 EFI loader
- transitional BootInfo v2-compatible loader fact model
- kernel image loading and verification path

Acceptance:
- boots in QEMU Arm64 UEFI and on Spark
- cleanly exits boot services
- reaches kernel entry
- carries display/storage/accelerator facts forward without guessing

## M3: Arm64 kernel core

Deliverables:
- Arm64 exception vectors
- early page tables
- physical page allocator
- framebuffer text console
- panic path

Acceptance:
- kernel prints platform identity
- kernel halts cleanly after initialization
- the same hello path is testable in QEMU and on Spark

## Execution rule

Do not move to later milestones until each earlier one has a reproducible artifact and a short acceptance note.
